## Permitted Uses

### ✅ Allowed Without Permission

1. **Internal Development**
   - Use glyphs in applications you're building
   - Include in UI mockups and prototypes
   - Reference in technical documentation

2. **Educational Content**
   - Teaching materials about the MirrorDNA ecosystem
   - Academic research and publications
   - Conference presentations and demos

3. **Integration & Compatibility**
   - "Works with MirrorDNA" badges on compatible products
   - API integration indicators
   - Technical compatibility displays

4. **Open Source Projects**
   - Using glyphs to indicate MirrorDNA compatibility
   - Including in documentation
   - Tool integration markers

### ❌ Requires Written Permission

1. **Company Branding**
   - Using MirrorDNA glyphs as your company logo
   - Incorporating glyphs into your brand identity
   - Suggesting official partnership or endorsement

2. **Product Names**
   - Creating products named "MirrorDNA [Something]"
   - Using protected names in product titles
   - Confusingly similar product naming

3. **Commercial Resale**
   - Selling glyph assets as standalone products
   - Licensing glyphs to third parties
   - Creating derivative glyph packs for sale

4. **Official Representation**
   - Claiming official affiliation with N1 Intelligence
   - Using glyphs to suggest endorsement
   - Representing yourself as an official partner

## Brand Glyph Usage

### MirrorDNA

**When to use:**
- Indicating MirrorDNA protocol compatibility
- Documentation of MirrorDNA features
- API integration markers

**How to use:**
- Maintain original colors or use monochrome
- Don't alter the DNA helix structure
- Ensure minimum size of 24px

**DON'T:**
- Use as your primary logo
- Modify the design significantly
- Suggest official endorsement

### LingOS

**When to use:**
- LingOS integration features
- Reflective dialogue capabilities
- Language processing indicators

**How to use:**
- Respect the speech bubble symbolism
- Maintain aspect ratio
- Use in context of linguistic features

**DON'T:**
- Repurpose for generic chat features
- Alter the reflective line element
- Use for non-LingOS dialogue systems

### Active MirrorOS

**When to use:**
- Product integrations with Active MirrorOS
- Memory persistence features
- Intelligence continuity displays

**How to use:**
- Preserve the mirror/reflection symbolism
- Maintain the pulse animation if using dynamic version
- Use for memory-related features

**DON'T:**
- Use for generic AI products
- Remove the reflection symbolism
- Suggest official Active MirrorOS status without authorization

### TrustByDesign

**When to use:**
- Compliance and safety features
- Governance dashboards
- Trust boundary indicators

**How to use:**
- Preserve the shield and checkmark elements
- Use in security/trust contexts only
- Maintain symbolic integrity

**DON'T:**
- Use for generic security features
- Alter the shield design
- Make unverified compliance claims

### AgentDNA

**When to use:**
- Agent personality systems
- Persistence schema implementations
- Identity continuity features

**How to use:**
- Keep the DNA + persona combination
- Use for agent-related features
- Maintain design integrity

**DON'T:**
- Use for non-agent systems
- Separate the DNA and persona elements
- Repurpose for generic identity features

### Glyphtrail

**When to use:**
- Interaction logging
- Lineage visualization
- Event trail displays

**How to use:**
- Preserve the connected node pattern
- Use for lineage/trail features
- Keep the path visualization

**DON'T:**
- Use for unrelated logging
- Alter the connection symbolism
- Repurpose for generic analytics

## Modification Rules

### ✅ Allowed Modifications

- Resizing (maintaining aspect ratio)
- Recoloring for theme consistency
- Converting to monochrome
- Adding shadows/effects for depth (minimal)

### ❌ Prohibited Modifications

- Distorting aspect ratios
- Removing or altering core symbolic elements
- Creating derivative glyphs that could cause confusion
- Combining with other brand logos
- Rotating or flipping in ways that alter meaning
- Adding text or graphics inside glyphs

---

## Quality Standards

### File Format

- **Primary:** SVG (vector)
- **Fallback:** PNG with transparency
- **Minimum resolution:** 16px × 16px
- **Maximum file size:** Keep reasonable (< 10KB for SVG)

### Visual Quality

- Maintain sharp edges
- Preserve symbolic clarity
- Ensure legibility at all sizes
- Test on light and dark backgrounds
